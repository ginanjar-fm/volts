## Setup

### 1. Connect Your Repository

1. Go to [vercel.com/new](https://vercel.com/new)
2. Import your GitHub repository
3. Vercel auto-detects the Next.js framework preset

### 2. Set Environment Variables

In the Vercel dashboard, go to **Settings → Environment Variables** and add all required variables:

```
DATABASE_URL=postgresql://...
AUTH_SECRET=your-random-secret
AUTH_URL=https://your-app.vercel.app
AUTH_GOOGLE_ID=...
AUTH_GOOGLE_SECRET=...
AUTH_GITHUB_ID=...
AUTH_GITHUB_SECRET=...
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...
OPENAI_API_KEY=sk-...
```

Set `AUTH_URL` to your production domain (e.g. `https://app.example.com`). If using a custom domain, configure it under **Settings → Domains** first.

### 3. Deploy

Click **Deploy**. Vercel runs `npm run build` automatically.

Every push to `main` triggers a new production deployment. Pull requests get preview deployments.

## Stripe Webhook

Create a new webhook endpoint in the [Stripe Dashboard](https://dashboard.stripe.com/webhooks) pointing to your production URL:

```
https://your-app.vercel.app/api/webhooks/stripe
```

Subscribe to the events your app handles (e.g. `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`).

Copy the webhook signing secret and set it as `STRIPE_WEBHOOK_SECRET` in Vercel.

## Custom Domain

1. Go to **Settings → Domains** in your Vercel project
2. Add your domain
3. Update DNS records as instructed
4. Update `AUTH_URL` to match the custom domain

## Notes

- The framework preset is auto-detected as **Next.js** -- no manual configuration needed
- Vercel automatically handles serverless function splitting and edge caching
- Use **Vercel Postgres** or an external provider like Neon/Supabase for the database
